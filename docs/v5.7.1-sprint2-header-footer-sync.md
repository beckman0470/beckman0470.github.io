## 判斷

- `templates/components/footer.html` 存在，但首頁輸出未保證一定使用它。
- `engine/homepage.py` 仍可能產生自己的 Header / Footer。
- `index.html` 是目前 GitHub Pages 實際使用的首頁，因此本 Sprint 同時提供修正後的 `index.html`。
- 新增 `engine/components.py`，讓後續 Build 可以使用同一套 Header / Footer component。

## QA

上傳後檢查：

- `index.html` 首頁 Footer 是否仍為深綠色四欄。
- Header 導覽是否沒有文字黏在一起。
- 之後若執行 `cms/build.py`，首頁也會經由 `engine.components.inject_header_footer()` 套用共用 Header / Footer。
